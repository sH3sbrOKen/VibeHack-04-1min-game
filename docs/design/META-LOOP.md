# META-LOOP — 平行时间线系统设计

## 核心卖点

**「一分钟」是入口，不是游戏本体。**

主线是每个人都经历过的处境——手机快没电，消息来不及回。它的作用是建立熟悉感和操作习惯。真正的游戏，从结算页开始。

每局结束，玩家可能滑入一条平行时间线。时间线有独立世界观、独立叙事、部分甚至有颠覆性的机制。玩家不知道有多少条，也不知道下一条是什么——发现本身就是奖励。

**终极目标：100个结局。** 参考《百日战纪》的设计思路：海量碎片化叙事，每条线短小精悍，整体拼出一个更大的世界。

## 设计支柱

### 1. 规则颠覆性
每条支线可以完全改变操作逻辑，不局限于"换一批发信人"。

| 机制类型 | 示例 |
|---------|-----|
| 正常回复（默认） | 骑手、职场危机、分手现场 |
| 完全不操作 | 垃圾短信线——回任何一条都会让骚扰变本加厉，唯一通关方式是60秒什么都不做 |
| 反向判断 | 诈骗话术线——你是诈骗分子，要判断哪条消息是卧底 |
| 时序依赖 | 宇航员线——回复顺序错误会让对方走向死亡 |

### 2. 情感钩子
高情感代价制造执念式重玩。玩家"救"不了宇航员，会反复重试。玩家发现某条线有隐藏结局，会专门刷数值去触发。

### 3. 渐进披露
- 玩家第一局不知道支线存在
- 第一次触发数值阈值，结算页出现「???已解锁」
- 进入支线后发现世界观完全不同，产生惊喜
- 后续通过不同的数值组合，发现更多入口

### 4. 碎片叙事
单条支线叙事碎片化，一局只展示一个片段。多次进入同一支线，随机抽取不同消息卡，叙事逐渐拼全。玩家第10次进入才意识到支线里的角色和主线有关联。

## 支线分级

### A 级——机制相同，世界观替换（开发成本低）
适合快速落地，先建立"有很多线"的感知。

- 骑手一分钟
- 职场危机一分钟
- 分手现场一分钟（等）

### B 级——机制微变（加时序/限制条件）
在现有引擎基础上加少量逻辑。

- 宇航员线：消息有时序依赖，回错了宇航员状态恶化（参考《生命线》）
- 客服投诉线：部分消息需要"组合回复"才能解决

### C 级——机制颠覆（需要新引擎逻辑）
开发成本最高，也是最强的惊喜感体验。

- 垃圾短信线：完全不操作才是通关条件
- 诈骗话术线：视角翻转，玩家是"坏人"
- 静默线：只有一条消息，60秒内要决定回还是不回

## 解锁逻辑

支线通过主线结算时的数值状态解锁。同一局可以同时满足多个解锁条件，结算页显示所有已解锁入口。

**随机触发机制（长期目标）**：部分支线不绑定数值，而是根据局内行为随机触发——比如连续3条消息全部点同一个按钮，触发隐藏线。这层不可预测性是"100个结局"能维持探索动力的关键。

## 实现路线图

**黑客松阶段（v0.6）**
- 实现支线入口框架：结算页解锁项旁加「进入」按钮
- 完成一条完整 A 级支线原型
- 结算页显示「时间线 #001」编号，暗示更多存在

**黑客松后（v1.0）**
- A 级支线全部可玩（#002~#007）
- 实现一条 B 级支线（宇航员 #008）
- 解锁地图 UI：显示玩家已发现/未发现的线

**长期（v2.0+）**
- C 级支线
- 跨支线叙事连接和彩蛋
- 向100个结局推进
