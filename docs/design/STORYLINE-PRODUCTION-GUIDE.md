## 一、项目背景

**游戏**：「今天不想回消息」——手机1%电，99+消息，60秒用四个预设按钮回复。
**目标**：12条故事线 × 每线 ~8-10 个结局 = **100个结局**。
**截止**：2026-05-30 18:00 路演。

### 核心概念

| 概念 | 定义 | 对玩家可见? |
|------|------|-----------|
| **故事线** | 一个独立的一分钟世界，有专属消息卡/数值/结局 | 是（结算页看到名称和结局） |
| **世界线** | 多条故事线之间的隐藏跳转链路，有明确的起点和终点 | 否（玩家只感受到"故事自然延续"） |

**故事线**可以有独特的进入条件（不限于数值阈值，可以是行为模式）和脱离条件（特殊通关方式）。

**世界线**定义了故事线之间的因果关系：未脱离故事线 A → 跳转到故事线 B；脱离了 A → 回到主线或跳转到 C。不是所有故事线都必须属于某个世界线，独立故事线只需定义进入条件即可。

## 四、文案风格（COPY-STYLE 摘要）

**无厘头冷淡幽默**——荒诞的场景用日常的语气说出来。

### 核心规则

- **短句优先**：每句话独立成行
- **不用感叹号**：去掉感叹号读起来反而更好
- **冷淡陈述荒诞事实**：不评价、不煽情，只陈述
- **括号里埋后记**：主体说完正经的事，括号里放一句反差极大的日常观察
- **结尾留白**：不总结，不升华，停在一个奇怪的地方

### 检查清单

- [ ] 无感叹号
- [ ] 无「终于」「居然」「竟然」等情绪词
- [ ] 最后一句停在奇怪的地方
- [ ] 括号内容比主体更日常、更小、更荒诞
- [ ] 读完找不到明确笑点，但想笑

### 反面示例

```
❌ 恭喜你！你回完了所有消息！太厉害了！
❌ 你以超凡的效率完成了不可能的任务。
```

#### 进入条件说明

进入条件不限于数值阈值，可以是任何可判定的玩家行为。策划用自然语言描述，程序转成代码。

| 类型 | 格式 | 示例 |

| 处理条数 | `{来源故事线} 正确处理 ≥/≤ N 条` | `主线 正确处理 ≤ 5 条` |
| 回复总数 | `{来源故事线} 回复 ≥/≤ N 条` | `主线 回复 ≥ 60 条` |
| 结局触发 | `触发过 {结局ID 或结局标题}` | `触发过 RIDER_E02 超时大王` |
| 世界线跳转 | `由世界线 {世界线ID} 指定` | `由世界线 SCAM_CHAIN 指定` |
| 特殊行为 | 自由描述 | `连续5条消息点了同一个按钮` |

#### 脱离条件说明

脱离条件是**在这条故事线内**玩家需要做到什么才能"通关脱离"。不填则无特殊通关条件（玩完60秒自然结算）。

**脱离状态影响世界线走向**：脱离了 → 走世界线的脱离分支；未脱离 → 走未脱离分支。

| 类型 | 示例 |
|------|------|
| 完全不操作 | `60秒内完全不点击任何按钮` |
| 达成数值 | `本线 TRUST ≥ 10` |
| 行为模式 | `连续正确回复 ≥ 10 条` |
| 特殊操作 | `最后一条消息选择了 LOVE` |
| 自由描述 | 用自然语言写，程序会实现 |

## 七、世界线定义（集中在 WORLDLINES.md）

### 什么是世界线

世界线是一组故事线之间的**隐藏因果链**。玩家不知道世界线的存在，只会感受到"上一局的选择影响了下一局的去向"。

### 世界线结构

每条世界线需要：

| 字段 | 说明 |
|------|------|
| **世界线ID** | 大写英文，如 `SCAM_CHAIN` |
| **世界线名**（内部用） | 如「诈骗深渊线」 |
| **起点故事线** | 世界线从哪条故事线开始 |
| **终点故事线** | 世界线走到哪条故事线结束 |
| **跳转规则** | 每个节点的脱离/未脱离分支各去哪里 |

### 跳转规则格式

用箭头图描述，程序会转成代码。**每个节点可以有多条脱离分支**，用脱离条件的名称区分：

```
故事线A
  ├── 未脱离 → 故事线B
  ├── 脱离-{分支名1} → 回到主线
  └── 脱离-{分支名2} → 故事线C
```

- **未脱离**：60秒结束，脱离条件未触发
- **脱离-{分支名}**：分支名对应故事线脱离条件里定义的具体方式（如「60秒不操作」「全部退订」）。分支名由文案自由命名，程序对应实现

没有特殊脱离条件的故事线只需写"未脱离"和"脱离"两条。

### 世界线模板（写在 WORLDLINES.md 中）

````markdown
## {世界线ID} — {世界线名}

**起点**：{起点故事线ID}
**终点**：{终点故事线ID}

### 跳转规则

{起点故事线ID}
  ├── 未脱离 → {故事线ID} / 主线
  ├── 脱离-{分支名} → {故事线ID} / 主线
  └── 脱离-{分支名} → {故事线ID} / 主线

{下一故事线ID}
  ├── 未脱离 → {故事线ID} / 主线
  └── 脱离 → {故事线ID} / 主线
````

### 示例（SCAM_CHAIN）

## 十、设计参考

### 数值设计

- 一局60秒，玩家通常回 20-40 条消息
- 答错率约 30-50%
- ↑坏数值推荐阈值 3-8
- ↓坏数值推荐初始100、阈值 0-20
- 每个故事线的4个数值应覆盖不同维度
- 显示名越无厘头越好

### 结局设计

- 每个结局 2-3 行，最后一行最好是括号后记
- 数值结局：聚焦单一数值的荒诞后果
- 组合结局：两件事同时发生的混乱
- 特殊结局：奖励极端玩法（全对/全错/触发脱离条件等）
- 兜底结局：「什么都没发生」的平淡感

### 世界线设计

- 世界线是"如果你一直做错下去会怎样"的因果链
- 起点通常是一个行为模式触发的故事线（如错误回应占比高）
- 未脱离 = 继续深陷 → 通向更极端的故事线
- 脱离 = 醒悟 → 回到主线或走向另一个分支
- 世界线长度建议 2-4 个故事线节点

## 十一、验收标准

### 每条故事线

- [ ] 有4种以上发信人类型，4种回复按钮都被用到
- [ ] 有4个数值，↑坏和↓坏方向至少各1个
- [ ] 手写卡至少5张，其中至少1张陷阱卡
- [ ] 每种发信人类型至少2条消息模板
- [ ] 结局至少8个（4数值 + 2组合 + 1特殊 + 1兜底）
- [ ] 所有文案通过 COPY-STYLE 检查
- [ ] 开场旁白 2-4 行
- [ ] 进入条件明确（数值/行为/世界线指定）
- [ ] 如有脱离条件，描述清晰可判定
- [ ] 文件名与 ID 一致

### 世界线（WORLDLINES.md）

- [ ] 每条世界线有明确的起点和终点
- [ ] 跳转规则覆盖了每个节点的脱离/未脱离两个分支
- [ ] 引用的故事线 ID 都有对应的 STORYLINE 文件
- [ ] 起点故事线有独立的进入条件（不能只靠世界线自引用）
