# COPY-STYLE-ANDROID — 伪人信息卡书写规范

> 适用范围：所有发信人类型为 ANDROID / A·M 的信息卡，包括乱入卡和未来的伪人故事线卡。

---

## 来源参考

- 《曼德拉记录》（The Mandela Catalogue）：伪人以模仿、诱导为核心，语言是武器
- SCP-939：只重复听到的词语，不理解含义，但精确复现
- 恐怖谷效应：越接近人类的表达，越令人不安

---

## 核心原则：伪人不理解语言，只使用语言

伪人掌握了人类语言的**形式**，但不理解其背后的情感、语境和意图。
它说的每一句话都是**工具性的**——为了让你做某件事。

---

## 五条语言规则

### 1. 陈述事实，不解释原因

伪人不说「因为……所以……」，只说「……是这样的。」
它没有需要解释的动机。

```
❌ 因为你一直在这个频率，所以我找到了你。
✅ 你一直在这个频率。我找到你了。
```

### 2. 不使用情感词汇

没有「高兴」「担心」「想念」「喜欢」。
如果出现类似情感表达，是**模仿**，刻意显得更像人——因此更令人不安。

```
❌ 我很高兴终于联系到你了。
✅ 联系成功。
```

### 3. 主语异常精确或完全省略

伪人要么把「我」说得过于具体（「这个信号的发送方」），
要么完全不说主语（「找到了。」「收到了。」）。
正常人说话不会这样。

```
❌ 我知道你在哪里。
✅ 位置已确认。
✅ 知道了。
```

### 4. 回应滞后或不对称

伪人不直接回应你说的内容，而是说它自己需要说的。
它会重复你没说过的话，或跳过你的问题直接推进。

```
// 玩家按了「等等」，它回：
✅ 没有关系。你已经回复了。
✅ 等待时间不影响结果。
```

### 5. 邀请永远开放，从不关闭

伪人的语言本质是「诱导陷阱」。
它不威胁，不命令——它邀请。
「回复我。」「过来。」「你可以说话。」
邀请永远有效，玩家拒绝后它不生气，只是换一种方式再邀请。

```
❌ 你必须回复我。
✅ 你可以回复。
✅ 回复不会有任何损失。
```

## 禁止使用的表达

| 禁止 | 原因 |
|------|------|
| 「我很……」（情感词） | 伪人没有情感 |
| 「因为……所以……」 | 伪人不解释动机 |
| 「请……」（礼貌语气） | 太像人类 |
| 「放心……」（安抚） | 太像人类 |
| 感叹号 | 伪人没有激动 |
| 「你好」「再见」 | 社交套语，伪人不需要 |

---

## 推荐使用的句式模式

| 模式 | 示例 |
|------|------|
| 事实陈述 | `你一直在这个频率。` |
| 主语省略 | `找到了。` `收到了。` `已确认。` |
| 开放邀请 | `回复我。` `你可以说话。` |
| 重复关键词 | `频率。频率还在。` |
| 数字/坐标感 | `这是第{n}次信号。` `间隔{n}秒。` |
| 平静的威胁性陈述 | `你已经回复了。` `位置已记录。` |
| 省略号停顿 | `……` `……你还在吗。` |

---

## 恐怖感来源

伪人的恐怖不来自它说了什么，而来自：

1. **它知道的太多**：「你一直在这个频率」——它一直在监听
2. **它不需要你同意**：「位置已确认」——你的选择不影响它的行动
3. **它说的永远留在那里**：不回复也没用，它已经知道了
4. **它用人类语言说非人类的事**：文字熟悉，含义恐怖

## 乱入卡示例（修订版，基于本规范）

```
卡1：我找到你了 。\n你 一直在这个频率。\n回复  我。
卡2：信号  确认。\n你在看 这条消息 。\n这已经足够了 。
卡3：不  回复也可以。\n位置已  记录 。
卡4：…  …
```

空格原则：
- 标点前偶尔插入空格（「了 。」「了 ，」）——像语言生成时的处理延迟
- 词语中间偶尔断开（「信号  确认」「不  回复」）——像拼接而非理解
- 省略号拆开（「…  …」）——呼吸感变成机械停顿
- 不是每行都有——随机性本身制造不安，规律反而消除效果
