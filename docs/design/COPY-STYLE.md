# COPY-STYLE — 文案风格规范

> 项目所有面向玩家的文字，包括弹窗、结算、开场、支线旁白，均遵守本文件。

---

## 一、风格定义

**无厘头冷淡幽默**

荒诞的场景用日常的语气说出来。
不渲染，不解释，不收束。
读完想笑，但找不到笑点在哪里。

---

## 二、核心原则

**短句优先**
每句话尽量独立成行。不用长句叠加情绪。

**不用感叹号**
感叹号会破坏这种语气。去掉感叹号读起来更好，说明方向对了。

**冷淡陈述荒诞事实**
不评价、不煽情，只陈述。让读者自己感受荒诞在哪里。

**括号里埋后记**
括号是这套文风最重要的技法。
主体说完一件正经的事，括号里轻轻放一句反差极大的日常观察。
往往是全段最好笑、最有记忆点的地方。

**结尾留白**
不做总结，不升华，停在一个奇怪的地方。

## 三、已有文案示例

### 恭喜弹窗（回完99条提前结算）
```
你回完了
大概是今年最高效的一分钟

99条消息，1%的电。
每一条，你都没有放弃。

（外卖应该凉了。）
```

### 主页一句话
```
手机1%电，99+消息，四个字救不了所有人。
```

### 支线关卡开场旁白

骑手一分钟：
```
单量第38单。导航又在转圈。楼道里没有灯。
回复那些等着饭的人——你只有四句话。
```

职场危机一分钟：
```
会议室里，五个人都在看你。
你的手机也快没电了。
只是这次，你是老板。
```

分手现场一分钟：
```
你知道你说错了什么。
你不知道怎么说对。
还有六十秒，消息在一条条进来。
```

骚扰接线员一分钟：
```
这份工作的考核指标只有一个：
判断对方是真人还是机器。
前三十秒你以为你做到了。
```

## 四、写新文案时的检查清单

- [ ] 去掉感叹号，读起来反而更有力
- [ ] 没有「终于」「居然」「竟然」之类的情绪词
- [ ] 最后一句停在一个奇怪的地方，没有收束
- [ ] 如果有括号，括号里的内容比主体更日常、更小、更荒诞
- [ ] 全文读完，找不到明确的笑点，但就是想笑

---

## 五、反面示例（不要这样写）

```
❌ 恭喜你！你回完了所有消息！太厉害了！
❌ 你以超凡的效率完成了不可能的任务。
❌ 感谢你的努力，你真的很棒！
```

原因：有感叹号，有夸奖，有情绪，有收束。全错。
