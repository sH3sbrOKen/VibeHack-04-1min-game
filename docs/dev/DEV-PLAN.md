## 一、技术选型

**纯网页游戏（HTML + CSS + JavaScript）**

选择理由：
- 无需安装任何开发环境，浏览器直接运行
- AI可以整段生成代码，直接复制粘贴验证
- 黑客松当天分享链接，投票用户直接在手机上玩
- 手机竖屏适配简单，CSS即可实现

**不使用任何框架**，保持代码简单，AI报错后容易修。

## 三、Claude Code 提示词模板

### 阶段1：搭框架

```
用HTML+CSS+JavaScript做一个手机竖屏小游戏。

游戏规则：
- 屏幕顶部显示60秒倒计时和"电量1%"
- 消息从屏幕中央弹出（一次显示一条），显示发信人姓名和正文
- 屏幕四个角各有一个按钮：左上"等等"、右上"收到"、左下"退订"、右下"好的亲"
- 点击任意按钮，消息消失，弹出下一条
- 60秒结束后停止弹出消息，显示"游戏结束"

先只实现这个基础框架，消息内容先用3条测试数据即可。
```

### 阶段2：填入判断逻辑

```
在现有代码基础上，增加答对/答错判断逻辑：

每条消息有一个正确答案（WAIT/RECEIVED/UNSUBSCRIBE/LOVE对应四个按钮）。
玩家点击按钮后：
- 答对：消息正常消失
- 答错：消息消失 + 对应数值发生变化（先用console.log打印出来）

数值列表：
- DELIVERY_LOST（外卖丢失率，初始0，答错+1）
- BOSS_PATIENCE（老板耐心值，初始100，答错-15到-30）
- INFO_LEAK（个人信息泄露指数，初始0，答错+1到+2）
- INTIMACY（亲密度，初始100，答错-5到-30）
- SPAM_SUBSCRIBED（骚扰订阅量，初始0，答错+1）
- UNEXPECTED_CHARGE（意外扣费金额，初始0，答错+12）
```

### 阶段3：加入完整信息卡

```
把以下信息卡数组填入游戏代码，替换测试数据：

[粘贴CONTENT.md中的信息卡，转换为JS数组格式]

格式参考：
const messages = [
  { id: "MSG_001", senderName: "老板", body: "今天的日报发了吗，我没收到", correctReply: "RECEIVED", wrongEffect: { BOSS_PATIENCE: -15 } },
  ...
]
```

### 阶段4：数值视觉反馈

```
当数值发生变化时，在屏幕上显示飘字效果：
- 红色字体，从点击位置向上飘动后消失
- 内容示例："外卖丢失率 +1" / "老板耐心 -15"
- 动画时长约0.8秒

同时在屏幕角落显示当前数值（小字，半透明）
```

### 阶段5：结算页面

```
60秒结束后，显示结算页面：
- 标题：「今天不想回消息」
- 展示6个数值的当前状态
- 达到触发阈值的数值，旁边显示解锁提示文字（见下方）
- 底部显示分享文案

解锁提示对应关系：
- DELIVERY_LOST ≥ 3 → 「骑手一分钟 已解锁」
- BOSS_PATIENCE ≤ 0 → 「职场危机一分钟 已解锁」
- INFO_LEAK ≥ 3 → 「电话销售一分钟 已解锁」
- INTIMACY ≤ 0 → 「分手现场一分钟 已解锁」
- SPAM_SUBSCRIBED ≥ 5 → 「骚扰接线员一分钟 已解锁」
- UNEXPECTED_CHARGE ≥ 200 → 「客服投诉一分钟 已解锁」
```

## 四、遇到问题时的处理方式

### 代码报错
直接把报错信息粘贴给 Claude Code：
> 「运行这段代码出现了以下报错，帮我修复：[粘贴报错]」

### 效果不对
描述看到的和期望的差异：
> 「现在消息弹出的位置在屏幕底部，我希望它在屏幕中央偏上。帮我调整。」

### 功能不理解
问清楚再让AI做：
> 「我想让消息弹出时有个从下往上滑入的动画，你打算怎么实现？」

### 卡住3次没进展
停下来，把卡住的描述发到这个对话，重新拆分任务。
