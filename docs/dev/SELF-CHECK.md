# SELF-CHECK — 功能自检清单

> 触发时机：① 每次完成新功能开发后自动执行；② 用户说「自检」或「复盘」时执行。
> 结果三级：✅ 通过 / ⚠️ 疑似问题（向用户说明）/ ❌ 确认冲突（必须先修再报告完成）

---

## 检测维度

**[SC-001] 因果自洽性** ← A-001
检测对象：信息卡的 `wrongEffect`、新增数值触发逻辑
检测方法：玩家回错这张卡，在现实生活中真的会造成该效果吗？
- 反例：银行消费通知答错 → 意外扣费（通知不影响是否扣费）
- 正例：外卖没等 → 外卖丢失率 +1（合理因果）

---

## 自检报告格式

```
自检完成：[功能名]
SC-001 因果自洽：✅/⚠️/❌ [一句说明]
结论：通过 / 发现 X 个问题：[列出]
```
