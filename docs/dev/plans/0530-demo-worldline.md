# 0530-demo-worldline — 路演演示世界线

## 目标

1. 开始页新增「演示模式」按钮，直接启动路演世界线
2. 路演世界线：故事线顺序固定，自动串联，无需玩家触发解锁
3. 游戏中新增「跳过→结算」按钮，随时可跳到当前故事线结算
4. 世界线内所有故事线顺序由文案配置，程序读取

## 阻塞

**缺文案交付**：故事线顺序未定。需要文案填写 `DEMO_WORLDLINE_CONFIG`（见配置文档）后才能实现第1、2条。

第3条（跳过按钮）不依赖文案，可先实现。

## 改动范围（仅 index.html）

### 1. 跳过按钮（不依赖文案，先做）
- HTML：游戏区新增 `#skip-btn`，绝对定位，游戏中显示，结算时隐藏
- JS：点击 → `clearInterval(tik)` → `endGame()`
- CSS：低调样式，不干扰主界面

### 2. DEMO_WORLDLINE 配置（等文案）
```js
const DEMO_WORLDLINE = {
  sequence: ['MAIN', 'SCAM_TARGET', 'DATA_AUCTION'], // 文案填写
  autoNext: true,  // 当前故事线结束自动进入下一条
};
```

### 3. 演示模式启动（等文案）
- 开始页新增「▶ 演示模式」按钮
- `startDemoMode()` → 初始化序列指针 → `startGame(sequence[0])`
- `endGame()` 中检测 demo 模式 → 自动渲染「下一段」按钮替代「再来一局」

### 4. 演示序列推进
- `demoIndex` 全局变量记录当前位置
- 结算页「下一段」→ `startGame(sequence[++demoIndex])`
- 到达末尾 → 显示「演示结束」+ 返回主线按钮

## 不做的
- 演示模式的结局判定（用正常结局系统，不特殊处理）
- 强制解锁演示序列中的故事线（演示模式绕过解锁判定直接进入）

## 验证
1. 开始页有「演示模式」按钮
2. 点击后直接进入第一条故事线（无开场画面延迟）
3. 游戏中右下角有「跳过」按钮，点击直接结算
4. 结算页有「下一段」按钮，点击进入下一条故事线
5. 最后一条结束后显示「演示结束」
